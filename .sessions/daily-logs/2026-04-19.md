### Windows-Specific Fixes (not in plan — discovered during execution)

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| `next/font/google` fails | `ERR_UNSUPPORTED_ESM_URL_SCHEME` — Node ESM loader can't resolve `C:` paths on Windows | Removed `next/font/google` from layouts |
| `postcss.config.mjs` fails | Same ESM URL issue with PostCSS loader | Replaced with `postcss.config.js` (CJS) |
| `output: 'standalone'` EPERM | Windows requires Developer Mode for symlinks | Conditioned on `NEXT_OUTPUT=standalone` env var |
| shadcn v4 + Tailwind v3 | `shadcn@latest` now generates Tailwind v4 syntax (oklch, `@import "shadcn/tailwind.css"`) incompatible with v3 | Rewrote `globals.css` with HSL CSS vars; replaced `button.tsx` to use plain HTML (`@base-ui/react` is v4 only) |
| openapi-fetch + placeholder types | Placeholder `paths = Record<string, never>` makes all API calls typed as `never` | QR verify page uses `fetch()` directly; typed client works once codegen runs |
