### Bugs Found and Fixed

#### Bug 1 — `GET /products/export` returns 500 (scenario-11 Step 11.7)

**Root cause:** `exportProductRegistry` in `src/modules/product/services/product.service.ts` referenced `p.regionCode` and `p.status` — neither column exists on the `Product` entity (entity only has: id, cooperativeId, productTypeCode, name, description, createdAt, updatedAt, createdBy, deletedAt).

**Fix:**
- Added `import { ProductType }` to product.service.ts
- Replaced `.addSelect('p.regionCode', 'regionCode')` with `.leftJoin(ProductType, 'pt', 'pt.code = p.productTypeCode').addSelect('pt.regionCode', 'regionCode')`
- Removed non-existent `.addSelect('p.status', 'status')` and the `status` CSV column
- Updated the unit test mock in `test/unit/product/product.service.spec.ts` to add `leftJoin: jest.fn().mockReturnThis()` to `makeExportQb`

**Files changed:**
- `src/modules/product/services/product.service.ts`
- `test/unit/product/product.service.spec.ts`

#### Bug 2 — `cooperative-onboarding.e2e.ts` flaky due to missing cleanup

**Root cause:** Test created a cooperative (ICE `001234567000099`) but had no `beforeAll` cleanup. When `--forceExit` killed a previous run before `afterAll` could delete the cooperative, the next run's `POST /cooperatives` returned 409 (conflict), causing `GET` to receive undefined ID → 404.

**Fix:** Added `DataSource` import and `DELETE FROM cooperative.cooperative WHERE ice = $1` in both `beforeAll` (pre-seed guard) and `afterAll` (explicit cleanup).

**Files changed:**
- `test/e2e/smoke/cooperative-onboarding.e2e.ts`

#### Bug 3 — `scenario-2-cooperative-admin.e2e.ts` flaky due to missing beforeAll cleanup

**Root cause:** Scenario-2 `beforeAll` seeded cooperative with `ON CONFLICT DO NOTHING` but did NOT clean up existing members/farms first. After a `--forceExit` killed a previous run's `afterAll`, member Hassan Oubella (CIN `J123456`) persisted. Next run's `POST /cooperatives/:id/members` returned 409 (MEMBER_DUPLICATE_CIN) instead of 201. 3 tests failed.

**Fix:** Added pre-seed cleanup in `beforeAll`:
```ts
await ds.query(`DELETE FROM cooperative.farm WHERE cooperative_id = $1`, [S2_COOP_ID]);
await ds.query(`DELETE FROM cooperative.member WHERE cooperative_id = $1`, [S2_COOP_ID]);
await ds.query(`DELETE FROM cooperative.cooperative WHERE id = $1`, [S2_COOP_ID]);
```

**Files changed:**
- `test/e2e/scenario-2-cooperative-admin.e2e.ts`

## Key Decisions

1. **beforeAll cleanup pattern** — Any e2e test that seeds data should DELETE the data in `beforeAll` (before INSERT) to guard against `--forceExit` killing `afterAll`. This is now the established pattern.
2. **Product export uses leftJoin not raw SQL** — Kept `createQueryBuilder` approach so existing unit test mocks stay valid; only added `leftJoin(ProductType, ...)`.
3. **Dropped `status` from product export CSV** — `Product` entity has no `status` column; the column was silently wrong. CSV now has 6 columns: productId, name, productTypeCode, cooperativeId, regionCode, registeredAt.
