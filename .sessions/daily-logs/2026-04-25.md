## Decisions Made

1. **Listener spec updates pulled forward** — typecheck gate failed at Batch 1 because existing specs still passed `{} as never` second args; fixed immediately rather than waiting for Task 6
2. **All 4 listeners now live** — KafkaClientModule left in place (modules still import it) but the old `@EventPattern` route is gone; all actual event handling goes through `KafkaConsumerService`
3. **terroir-portal client added (not web-portal modified)** — design already had `terroir-portal` in `.env.example`; cleanest fix is to add the matching client to the realm rather than rename `web-portal`
4. **Browser-based Playwright auth** — no `directAccessGrantsEnabled` needed; flow goes through NextAuth → Keycloak login form → redirect back; simpler and more realistic than direct grant
5. **Smoke assertions only** — full journey tests (inspector files report → cert-body grants) deferred; smoke (page renders, not 500, not login redirect) is correct for this sprint
6. **global-setup writes a seed.json stub** — no API seeding done; smoke tests don't need specific DB records; the stub allows `qr-verify.spec.ts` to skip gracefully when .auth/ absent
