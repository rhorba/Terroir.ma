### Decisions

| Decision | Rationale |
|----------|-----------|
| RSC for all list/detail + Server Actions for mutations | Consistent pattern across all FE sprints; no client state needed for admin listings |
| `apiFetch<T>` in `api-server.ts` wraps `fetch()` directly | api-client migration deferred to FE-S8 when codegen runs against live API |
| `randomUUID()` from Node `crypto` for `x-correlation-id` | Avoids adding `uuid` package dependency |
| Super-admin home `try/catch` around dashboard fetch | Never crashes layout when backend is offline during dev |
| Sidebar nav split into 'Paramètres' + 'Journal d'audit' | Two separate concerns, cleaner navigation |
| `gh` CLI installed via `winget` | No `choco` or `scoop` available; `winget` v1.28 was present |
| Two separate device auth flows for `gh` | Initial login (repo scope) + refresh (workflow scope) — GitHub OAuth Apps require explicit scope grant |
